# Examples

## Paper Interpretation

```text
Use top-tier-research-reviewer to interpret this paper. Separate claimed innovation, actual innovation, engineering combination, transferable macro ideas, and details not worth copying.
```

## Manuscript Review

```text
Review this manuscript as a top-tier journal reviewer. Give fatal issues, major issues, minor issues, publication-readiness level, and a directly executable revision checklist.
```

## Innovation Design

```text
Given this dataset and research scenario, help me design a top-tier research direction. Do not begin with a neural network. Start from the real contradiction, uncertain variables, unrealistic assumptions, error propagation, and appropriate research paradigm.
```

## Experiment Design

```text
Build a contribution-experiment-metric matrix for this proposed method. Check whether the current experiments include strong baselines, ablations, sensitivity analysis, multiple random seeds, robustness tests, extreme scenarios, and leakage checks.
```

## Code-Paper Consistency

```text
Audit this repository against my manuscript. Check feature consistency, variable/equation implementation, train-validation-test split, scaler fitting, future information, loss functions, model dimensions, ablations, and result-table consistency.
```

## Rebuttal And Revision

```text
Analyze these reviewer comments. Cluster the concerns, identify fatal vs fixable issues, propose additional experiments, and draft a point-by-point rebuttal.
```

## PPT Compression

```text
Compress this method into one PPT slide. Give a slide title, one-sentence goal, 3-5 core blocks, 6-10 step flow, formulas to keep, details to remove, figure layout, and a 60-90 second oral script.
```

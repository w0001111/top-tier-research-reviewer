# PPT Compression Rules

## One-Slide Rule

A slide should communicate one central message only.

Keep one objective statement, 3–5 core modules, 6–10 process steps, at most 1–2 key formulas, one compact figure or flow diagram, and one takeaway sentence.

Remove long pseudocode, implementation details, secondary variables, repeated definitions, formulas not used in oral explanation, and table columns that do not support the main claim.

## Algorithm Compression

Convert code-like algorithms into system-level transitions: state observation, prediction/inference, optimization/decision, feasibility check, coordination/correction, execution, feedback update.

## Output Format

1. Slide title
2. One-sentence message
3. Figure layout
4. Block labels
5. Compressed process
6. Formula(s) to keep
7. Text to delete
8. 60–90 second oral script
